Its purpose is to avoid writing libraries for every language used on the platform. By default it listens lo the loopback interface, making IPC overheads very low. All user-facing services can use it natively (HTTP/JSON) without having to be SOAP clients and its API better matches callers' needs, thus it requires little to no wrapper and is used as a library. But as it is a service you can easily have more than one version running at the same time making upgrades and migrations, even partial and long ones, easier.
